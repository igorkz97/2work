## Команды Linux
1. `pwd` -Print Working Directory в каком каталоге находимся
2. `ls`  -позволяет просмотреть содержимое каталога   
3. `cd` <> -переход в другую директорию `cd /home/student/log`  (абсолютный путь)
4. `cd log`  --- перейди в папку log из той директории, где ты сейчас находишься (Относительный путь — относительно вашей текущей директории, где вы сейчас находитесь)
5. `cd ..` -подняться на уровень выше
6. `cd ../..` -подняться на два уровня выше
7. `mkdir test`  --- создает папку с названием «test» там, где вы находитесь
8. `touch app.log` Такая команда создаст пустой файл с названием «app.log». А потом уже можно открыть файл в редакторе и редактировать.
9. `nano` -создаёт текстовой документ и открывает. Указываете имя команды и путь в файлу: `nano test_env.json`
10. `Ctrl+O` — чтобы сохранить
11. `Ctrl+X` — для выхода
12. `vim` test_env.json -создаёт как `nano` `:q` — закрыть редактор,`:q!` — закрыть редактор без сохранения,`:w` — сохранить файл;
13. `cp` -что_копировать куда_копировать Команда `cp` `app.log /home/olga/app_test_2020_03_08.log` используется в операционной системе Linux для копирования файла.
Она копирует файл app.log в директорию /home/olga/ и одновременно переименовывает его в app_test_2020_03_08.log. Это удобно для создания резервных копий файлов или их переноса в другое место с изменением имени.
13. `mv` - мув перемещение файла
14. `rm` test.txt  — удалит файл test.txt , `rm *`  — удалит все файлы в текущей директории
15. Если нужно удалить папку, надо добавить флаг `-r` (recursive): `rm -r` test_folder
16. `rm -rf` test_folder    --- просто все удалит без разговоров
17. `chown` testbase:testbase test.txt — сменить владельца файла
18. `chown -R` testbase:testbase user  — сменить владельца папки
19. `chkconfig test on` Она добавит службу в автозапуск.
20. `whereis` newman -показывает где находится приложение **newman**
21. `tar` `-cvzf` `archive.tar.gz` /home/test -создание архива(В данном примере мы упаковали директорию /home/test, внутри которой было две картинки — 502.jpg и 504.jpg.)
22. Для распаковки меняем флаг «c» на «x» и убираем «z»: `tar` `-xvf` `archive.tar.gz`
23. `c` — создать архив в linux `x` — извлечь файлы из архива `v` — показать подробную информацию о процессе работы (без него мы бы не увидели, какие файлики запаковались / распаковались) `f` — файл для записи архива `z` — сжатие
24. `history` — показать последние 1000 команд
25. `df` `-h` -сколько свободного места на диске
26. `du` `-sh` -сколько весит директория  `du` `-sh *`  `---` - с разбиением
27. `hostname -i` -ip компьютера
28. `ifconfig` — выведет кучу инфы, в том числе ваш внешний IP
29. `ip a` — аналог, просто иногда `Ifconfig` дает очень много результата
30. `cat /etc/*-release` - версия операционной системы
31. `uname -a` -версия ядра
32. `man` `ls` -как узнать как работает команда
33. `ls` `-h` -если  нет или не работает `man`
34. `mkdir -p test/dir--{000..100}` `touch test/dir--{000..100}/file-{A..Z}` - создание множество файлов и папок
35. `mkdir` — создать директорию `touch` — создать файл (или изменить существующий, но если файла с таким именем нет, то команда создаст новый, пустой)
А выражения в скобках играют роль функции, которая выполняется в цикле и делает ручную работу за вас:
{000..100} — пробежится по всем числам от 0 до 100
{A..Z} — пробежится по всем буквам английского алфавита от A до Z
